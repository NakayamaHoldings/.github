# 💫 Nakayama Holdings

**Enterprise AI, IIoT & self-hosted cloud infrastructure — engineered for manufacturing.**

This is the organizational home for the R&D and product work of Nakayama Iron Works Ltd. — where we design, build, and self-host the systems that power our operations and customer experience.

## 🏭 What we build

**🔭 The N-Global product family**
- **N-Support** — a single platform for employees to manage 500s of customer records, 100s of machine analytics streams and IIoT data, with simplified customer support through N-Messenger.
- **N-Messenger** — a real-time messaging socket server built in Node.js to handle live traffic from 1000s of customers placing machine-parts orders and contacting support.
- **N-Client** — a web + native iOS (Swift) app letting customers manage their machine analytics, reach support, and receive instant updates from Nakayama Iron Works.

**🤖 OmniX Search** — a self-hosted AI platform for semantic search over ~50,000 engineering drawing PDFs, now expanding into translation, NL-to-SQL analytics, document generation, and speech.

**📇 Meishio** — a React Native business-card manager.

**☁️ Nakayama Cloud Services** — our own private cloud, built on R&D into clustering multiple on-premise servers. It hosts 20+ applications: web apps, Node and Python services, APIs, and multiple self-hosted LLMs.

## 🎯 What we're exploring
On-prem / self-hosted LLM tooling and cloud infrastructure for manufacturing — RAG over technical drawings and CAD, real-time IIoT + messaging systems, agentic workflows, and multi-server GPU orchestration.

## 🧪 R&D focus
Scaling vector search over large document sets, improving Japanese-language recall in retrieval pipelines, and hardening self-hosted clustering for high-availability across on-premise servers. We're also working with data warehousing & BI (star schemas, ETL, dashboarding), agentic AI patterns, and the NVIDIA agentic stack.

## 💬 Ask us about
Deploying LLMs on-premise, real-time messaging at scale with Node.js sockets, building IIoT + customer-support platforms, and AWS (EC2, Lambda, Route 53, IAM, S3 and more).
